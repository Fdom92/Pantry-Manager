# PantryMind v4.7 — Umbrella Design

> Release-level design covering sequencing, branch model, gates and shared assumptions
> for v4.7. Three sub-spec docs cover the work in detail:
>
> - [PRO sub-spec](2026-06-05-v4.7-pro-design.md)
> - [Retention sub-spec](2026-06-05-v4.7-retention-design.md)
> - [Quality sub-spec](2026-06-05-v4.7-quality-design.md)

## Release identity

- **Codename**: "PRO viable + retention base"
- **Version**: 4.7 (`package.json` `"4.7"`, `versionCode 53`)
- **Platform**: Android only
- **Duration**: ~4 weeks, solo dev
- **Predecessor**: v4.6 (PostHog + Sentry + reconsent + in-app update prompt already shipped)

## Hypothesis

- PRO conversion: 0% → 1–3% within 2 weeks post-ship
- D7 retention: +1–2 percentage points
- MAU trend: from -7%/mo → stable
- APK size: -1.5 MB minimum

## Sub-spec structure

```
docs/superpowers/specs/
  2026-06-05-v4.7-umbrella-design.md   ← this file
  2026-06-05-v4.7-pro-design.md         ← waste tracker, reposition, trial, pricing
```

Each sub-spec gets its own implementation plan via `superpowers:writing-plans` and
ships behind its own sub-branch. The umbrella tracks the release as a whole.

## Sequencing inside 4 weeks

```
Week 0      Pre-flight: review PostHog + Sentry data from v4.6 (1 day)
Week 1-2    PRO sub-spec
            ├ Waste tracker
            ├ Reposition prediction
            ├ PRO trial 7d (RevenueCat config + client surfaces)
            └ Pricing visible in Settings PRO hero
Week 3      Retention sub-spec
            ├ Streak counter + grace logic
            ├ Display (Dashboard chip + Settings card)
            └ Milestones (in-app toast + +24h push)
Week 4      Quality sub-spec
            ├ ProGuard / minifyEnabled (highest risk → ships first within Quality week)
            ├ Lazy-load jspdf
            ├ Insights empty state
            └ Settings loading skeleton
```

Quality items with surface overlap (Insights empty state, jspdf lazy-load) MAY ride
along inside PRO PRs when convenient. ProGuard and Settings skeleton ship standalone
during Week 4.

## Pre-flight gate (Week 0)

Before opening PRO sub-spec work, spend 1 day reviewing v4.6 telemetry:

**PostHog**
- Funnel: app_open → onboarding_completed → first_pantry_add
- Insights tab opens (free vs PRO)
- `/upgrade` views
- Modal open events per modal type (denominators for engagement)

**Sentry**
- Crash-free sessions %
- Top 5 errors by frequency
- Top 5 errors by user impact

**Decision rules from review**
- Crash-free sessions < 99% → swap ProGuard week with a stability fix sprint
- Settings page never appears in slow-load percentiles → drop skeleton item, replace with another Quality candidate (or shorten Quality week)
- A surprise top error → spawn separate hotfix PR before v4.7 PRO work starts

This gate produces a 1-page memo `docs/superpowers/notes/2026-06-05-v4.7-preflight-notes.md`
that informs the final sub-spec plans. The gate does NOT change committed designs by itself —
if findings demand spec changes, re-run brainstorm flow on the affected sub-spec.

## Decision gates

| Gate | When | Decision |
|---|---|---|
| Pre-flight | Week 0 day end | Lock or amend Quality items per real data |
| Mid-release | End of Week 2 | If PRO sub-spec slips > 3 days → defer pricing visible item to Quality week or drop trial CTA from 2 surfaces |
| Post-ship | +2 weeks after rollout | PRO conv ≥ 1? D7 +1pp? MAU stable? APK -1.5MB? → continue to v4.8 plan |
| Post-ship fallback | +2 weeks after rollout | If PRO conv stays at 0 → spawn v4.7.1 mid-cycle: onboarding rework using v4.6 funnel + trial extended to 14d |

## Shared assumptions

- v4.6 telemetry stack (PostHog EU + Sentry EU) is the source of truth for measuring v4.7.
- No new backend work required for v4.7. Reposition + waste compute happens client-side
  from existing `PantryEvent` documents in PouchDB. Insights LLM endpoint untouched.
- RevenueCat 7-day intro offer is configured in the RevenueCat dashboard, not in code.
  Client work for trial = surface "Empieza prueba gratis" CTAs + event emission only.
- Streak persistence reuses PouchDB. No new infrastructure.
- All copy lands in 6 languages (es, en, de, fr, it, pt) per project convention.

## Out of scope (deferred to v4.8 per roadmap)

- LLM-backed smart notification copy
- Restock prediction notifications (push side of reposition)
- Custom alert rules
- Sankey diagram / heat map
- D1 activation push
- Onboarding tooltips contextual
- Yearly Wrapped

## Risk summary (release-level)

| Risk | Mitigation |
|---|---|
| 4-week solo budget overruns | Mid-release gate at Week 2 trims surfaces, not the core 4 PRO items |
| Pre-flight finds critical issue | Pre-flight memo can re-prioritise Week 4 Quality scope or trigger hotfix branch off `main` |
| PostHog conversion data invalid (events miswired) | Pre-flight gate validates v4.6 event firing in real prod data before relying on it |
| ProGuard breaks plugin reflection | Quality week sequenced ProGuard first → 4 days remain to revert/fix |
| Trial billing edge cases (RevenueCat) | `hasUnusedTrial` signal hides CTA when ambiguous; surfaces plain upgrade button instead |
