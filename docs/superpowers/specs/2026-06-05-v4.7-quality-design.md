## §1 ProGuard / `minifyEnabled true`

### Change

`android/app/build.gradle`:

```groovy
buildTypes {
    release {
        minifyEnabled true
        shrinkResources true
        proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
    }
}
```

### ProGuard rules file

New `android/app/proguard-rules.pro` with preserve rules for:

- **Capacitor core** — `BridgeActivity`, `@CapacitorPlugin` annotations, JSValue types,
  WebView bridge classes. Capacitor 7 publishes baseline rules; cross-check Capacitor
  release notes.
- **Capacitor plugins in use** (per `package.json`):
  - `@capacitor/local-notifications` ^7.0.5
  - `@revenuecat/purchases-capacitor` ^11.2.15
  - `@capacitor-community/in-app-review` ^7.1.0
  - PostHog Android (transitive via PostHog JS, but check if separate native dep)
  - Sentry Capacitor (already wires `@sentry/capacitor`)
  - In-app update plugin (shipped recently in `2470d0b feat(updates)`) — verify package
    name and add preserve rules.
- **Reflection-loaded model classes** — none expected in client code (Angular AOT
  eliminates reflection), but PouchDB-related Android shim may have some. Verify via
  release build smoke test.

### Validation

1. Build release APK with minify enabled.
2. Manual smoke test on physical device:
   - Launch (cold + warm)
   - Onboarding flow (slides 1-3, opt-in notifications, opt-in analytics, seed grid)
   - Add product (despensa + fresh)
   - Consume product
   - Schedule notification + observe permission flow
   - Trigger in-app review prompt (or use dev panel)
   - Trigger in-app update prompt (or use dev panel)
   - PostHog event emission (check live events in PostHog UI)
   - Sentry test crash from dev panel (verify it lands with readable stack)
   - PRO purchase flow (RevenueCat sandbox)
3. Measure APK size before/after. Target: ≥ 1.5 MB reduction.

### Sentry mapping upload

ProGuard renames classes/methods → Sentry stacks become unreadable without mapping.
Extend release script to upload mapping.txt via Sentry CLI:

```bash
sentry-cli upload-proguard \
  --org pantrymind \
  --project pantrymind-android \
  --version $VERSION \
  android/app/build/outputs/mapping/release/mapping.txt
```

Upload step is **non-optional** — block release if upload fails.

### Files touched (Section 1)

- `android/app/build.gradle` (modify)
- `android/app/proguard-rules.pro` (new)
- release script (locate via `scripts/` or root — extend with mapping upload)
- `.claude/DEV.md` (document new build + upload steps)
- `.claude/STATE.md` (note minify enabled + size delta)

### Rollback plan

If a plugin breaks at runtime → flip `minifyEnabled` back to false in same hotfix branch,
ship a v4.7.0.1 patch. Keep `proguard-rules.pro` checked in for next attempt.

## §2 Lazy-load `jspdf`

### Current state

`jspdf` ^3.0.4 listed in `package.json`. Imported at top level somewhere in
`features/settings/` or `core/services/export/` for PDF export functionality. Bundled
in main initial chunk (~250 KB gzip).

Investigation step: `grep -rn "from 'jspdf'" src/` to confirm exact location(s) before
editing.

### Change

Convert top-level import to dynamic import inside the export trigger function:

```ts
// before
import jsPDF from 'jspdf';

async exportPdf(): Promise<void> {
  const doc = new jsPDF();
  // ...
}

// after
async exportPdf(): Promise<void> {
  const { jsPDF } = await import('jspdf');
  const doc = new jsPDF();
  // ...
}
```

Angular 20 / esbuild handles chunk-splitting automatically for dynamic imports.

### Validation

- Build with `ng build --stats-json`.
- Run bundle analyzer (e.g. `webpack-bundle-analyzer` or `source-map-explorer`) on
  `main-*.js` before/after.
- Confirm `jspdf` no longer present in main chunk; lives in its own lazy chunk.
- Confirm PDF export still works on device.
- Measure initial bundle size delta (target ≥ 200 KB reduction).

### Files touched (Section 2)

- 1-2 files in the export service / module (identified via grep)

### Risks

- **No service worker installed.** Lazy chunk fetched on first export usage. If user is
  offline at that moment, export fails. Acceptable — PDF export is a rare action,
  retries online. Document in user-facing error toast.

## §3 Empty state for Insights tab

### Trigger conditions

Insights tab is "empty" when either:

- User has 0 pantry items, OR
- User has < 7 days of `PantryEvent` data — too thin for waste / reposition compute.

The Insights tab serves multiple PRO + free contexts. Empty state takes precedence over
all PRO teasers — if there's no data, there's nothing to tease.

### Design

- Illustration: existing SVG asset (reuse onboarding empty / pantry empty illustration
  if available; else simple bowl/pantry SVG).
- Heading: `insights.empty.title` — "Aún no hay insights".
- Body: `insights.empty.body` — "Añade productos a tu despensa para ver patrones de
  consumo, predicciones y desperdicio."
- CTA: "Ir a despensa" → `router.navigate(['/pantry'])`.

PRO users with no data see the same empty state. Free users with no data also see the
same empty state — no trial CTA, because tease without value is hostile.

### State

`InsightsStateService` gets a new computed signal:

```ts
readonly isEmpty = computed<boolean>(() => {
  const items = this.items();
  const events = this.events();
  if (items.length === 0) return true;
  const sevenDaysAgo = new Date(Date.now() - 7 * 86400 * 1000);
  const recentEvents = events.filter(e => new Date(e.timestamp) >= sevenDaysAgo);
  return recentEvents.length < 5;
});
```

Insights page template guards all sections with `*ngIf="!facade.isEmpty(); else empty"`.

### Files touched (Section 3)

- `features/insights/components/insights-empty-state/` (new)
- `features/insights/insights.page.html` (conditional render)
- `core/services/insights/insights-state.service.ts` (`isEmpty` computed)
- `src/assets/i18n/*.json` (~4 keys × 6)

## §4 Loading skeleton for Settings page

### Trigger

Settings page mounted, while any of:

- `RevenueCatService.offerings()` returns null
- `SettingsPreferencesService.preferences()` not yet loaded from PouchDB
- `CatalogOptionsService.catalog()` not yet hydrated

→ Show skeleton in place of real content.

Minimum display duration: 200ms. Avoids flicker on fast loads. Don't render skeleton at
all if all signals resolved synchronously within 100ms (use a `setTimeout(showSkeleton, 100)`
guard).

### Design

Skeleton matches Settings final structure:

- PRO hero block (large card placeholder with title bar + 2 lines + button bar)
- Preferences cards (3 placeholder rows)
- Language picker (1 row)
- About (1 row)

Uses Ionic 8's `ion-skeleton-text` with `animated="true"`. No new shadow / radius tokens.

### State

`SettingsStateService` (page-scoped, may already exist or live inline in
`settings.page.ts`) gets:

```ts
readonly isReady = computed<boolean>(() =>
  this.offerings() != null
  && this.preferences() != null
  && this.catalog() != null
);
```

Template:

```html
@if (!facade.isReady() && shouldShowSkeleton()) {
  <app-settings-skeleton />
} @else {
  <!-- real settings content -->
}
```

`shouldShowSkeleton` = signal toggled true 100ms after mount if `isReady` still false.

### Files touched (Section 4)

- `features/settings/components/settings-skeleton/` (new)
- `features/settings/settings.page.html` (conditional)
- `features/settings/settings.page.ts` (add `shouldShowSkeleton` signal + timing)
- `features/settings/` state service (extend with `isReady`)

## Architecture invariants

- All 4 items are mechanical. No new domain layer, no new services.
- ProGuard is native Android — highest revert risk. Ships first within Week 4 so 4 days
  remain to revert/fix if it breaks something undetected by smoke tests.
- jspdf lazy-load + Insights empty state may overlap with PRO PRs (both touch Insights
  tab area). Acceptable to fold into PRO PRs when convenient; otherwise standalone.
- Settings skeleton ships standalone.

## Risks

| Risk | Mitigation |
|---|---|
| ProGuard breaks a Capacitor plugin reflection path | Smoke test covers all 5 plugins; Sentry catches misses if released to internal track first |
| Sentry stacks unreadable post-minify | Mapping upload non-optional in release script — block release if upload fails |
| jspdf chunk fails to load offline on first export | Acceptable; rare action, retries online, user-facing error toast |
| Settings skeleton flashes on fast load | 100ms guard prevents flash; 200ms minimum display if shown |
| Insights empty state hides PRO trial CTA from new users | Intended — tease without value is hostile |

## Success gates

- APK size ≥ 1.5 MB smaller (release build)
- Initial JS bundle ≥ 200 KB smaller
- Sentry stack symbols still readable post-minify (verified via test crash)
- No new crash patterns appear in first 48h post-rollout
- Settings perceived-load metric improves (subjective; no instrumentation added)
