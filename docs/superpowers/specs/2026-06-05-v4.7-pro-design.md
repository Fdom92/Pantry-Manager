## §1 Waste tracker (PRO)

### Surfaces

- **Dashboard card** — new card "Has tirado N productos este mes" with breakdown by category. Free users see blurred placeholder + trial CTA.
- **Insights tab** — full waste section: trend (this month vs last), top wasted category, top wasted product, by foodType. Free users see same teaser pattern.

### Data source

`PantryEvent` documents with `eventType = 'EXPIRE'` in the last 30 days, grouped by
`categoryId` and `foodType`. Sum `quantity` as count. Do NOT compute euros or kg — per
brainstorm decision Q3 = D, ship as count only now; defer price + unit weight models to
v4.8 if user feedback demands.

### Domain layer

New pure-function module:

```
core/domain/insights/waste.domain.ts
```

```ts
export interface WasteSummary {
  windowDays: number;
  totalCount: number;
  byCategory: Array<{ categoryId: string; count: number }>;
  byFoodType: Array<{ foodType: FoodType; count: number }>;
  topProduct?: { productId: string; productName: string; count: number };
  previousWindowCount: number;  // same-length window immediately before, for trend
  trend: 'up' | 'down' | 'flat';
}

export function computeWasteSummary(
  events: ReadonlyArray<PantryEvent>,
  now: Date,
  windowDays: number,
): WasteSummary;
```

Zero Angular dependencies. Unit-testable with fixture event arrays.

### State

Extend `core/services/insights/insights-state.service.ts` (root-scoped, exists from v4.4):

```ts
readonly wasteSummary = computed<WasteSummary | null>(() => {
  if (!this.isPro()) return null;
  return computeWasteSummary(this.events(), new Date(), 30);
});
```

Dashboard pulls a view-model summary via existing dashboard state service.

### UI

- new `features/dashboard/components/waste-tracker-card/` — standalone Angular component
  - PRO state: count + top category chip + trend arrow
  - Free state: blurred numbers + lock icon + "Desbloquea con prueba 7 días" CTA
  - Empty state (PRO, count = 0): "¡Sin desperdicios este mes! 🎯"
- extend `features/insights/insights.page.html` with waste section

## §2 Reposition prediction (PRO)

### Math

```
velocity_per_day = sum(CONSUME.quantity in last 30d for productId) / 30
days_to_out = currentStock(productId) / velocity_per_day
```

Show prediction only when `velocity_per_day >= 0.05` (≈ 1 consume per 20 days) to filter
noise. `days_to_out` rounded to nearest day, capped at 90.

`confidence` field (`low | medium | high`) computed from sample size:
- < 3 consume events in window → `low` (or hidden)
- 3–9 events → `medium`
- ≥ 10 events → `high`

### Surface

Per brainstorm Q4 = B:
- **Dashboard card "Próximamente sin stock"** — top 3–5 items, days countdown.
- **Insights tab section** — full sortable list (by days asc, by category).

Free tier teaser: dashboard card shows item names blurred + count visible, exact days
hidden behind lock icon + trial CTA. Insights tab shows same pattern.

### Action

Tap item → calls existing `ListStateService.addItem(productId)` to append to shopping
list. Confirmation toast: "Añadido a la lista". Emits `repo_prediction_added_to_list`.

### Domain layer

```
core/domain/insights/reposition.domain.ts
```

```ts
export interface RepositionPrediction {
  productId: string;
  productName: string;
  categoryId?: string;
  currentStock: number;
  velocityPerDay: number;
  daysToOut: number;
  confidence: 'low' | 'medium' | 'high';
}

export function computeRepositionPredictions(
  items: ReadonlyArray<PantryItem>,
  events: ReadonlyArray<PantryEvent>,
  now: Date,
): RepositionPrediction[];
```

Pure. Excludes fresh items (state-based quantities, no velocity math applies).

### State

Extend `InsightsStateService`:

```ts
readonly repositionPredictions = computed<RepositionPrediction[]>(() => {
  if (!this.isPro()) return [];
  return computeRepositionPredictions(this.items(), this.events(), new Date())
    .filter(p => p.daysToOut <= 30);
});
```

### UI

- new `features/dashboard/components/reposition-card/`
- new `features/insights/components/reposition-list/`
- Insights tab gets a new dedicated section above existing AI insights (NOT a new tab segment — section pattern matches waste tracker placement)

### Empty / insufficient data state

If no predictions cross the velocity threshold (typical for new users): show
"Necesitamos más historial. Sigue usando la app un par de semanas." (PRO state)

## §3 PRO trial 7d

### RevenueCat configuration (out of repo)

Configure in the RevenueCat dashboard:
- Add 7-day free trial intro offer on existing monthly + annual subscription products.
- Eligibility: first-time subscribers per Apple/Google store rules. RevenueCat handles
  eligibility computation server-side.

### Client work

#### `RevenueCatService` extension

Add computed signal:

```ts
readonly hasUnusedTrial = computed<boolean>(() => {
  const offering = this.currentOffering();
  if (!offering) return false;
  const pkg = offering.monthly ?? offering.annual;
  return pkg?.product?.introPrice?.priceString != null
    && /* RevenueCat eligibility flag */;
});
```

Signal hides CTA when ambiguous (defensive: any reading uncertainty → show plain upgrade
button instead). Source field for eligibility comes from RevenueCat
`Purchases.getProducts().introPrice` + `Purchases.checkTrialOrIntroductoryPriceEligibility`.

#### New reusable component

```
shared/components/pro-trial-cta/pro-trial-cta.component.ts
```

- Input: `surface: 'upgrade_page' | 'settings_hero' | 'insights_tab' | 'waste_card' | 'reposition_card'`
- Renders: "Empieza tu prueba gratis de 7 días" button when `hasUnusedTrial()` is true,
  else falls back to "Hazte PRO" linking to `/upgrade`.
- Click handler:
  1. Emit `pro_trial_cta_clicked` with `surface`.
  2. Call existing `RevenueCatService.purchasePackage(pkg)`.
  3. On success → emit `pro_trial_started`.
- Dismissible per-session: surface holds an in-memory signal map keyed by `surface`.
  Dismissal lives in the existing `RevenueCatService` (or a sibling `ProCtaUiState`
  root-scoped service) as a `Set<surface>` signal. NOT persisted to localStorage —
  resets on every cold start so users see CTAs again next session.

#### Trial expiry detection

Background check in `AppComponent` resume:
- If localStorage flag `PRO_TRIAL_STARTED_AT` exists AND `RevenueCatService.isPro()` flipped
  from true → false → emit `pro_trial_expired` once (idempotent via
  `STORAGE_KEYS.PRO_TRIAL_EXPIRED_FIRED`).
- Flag set on `pro_trial_started` event.

## §4 Pricing visible in Settings PRO hero

### Change

Settings PRO hero currently shows generic copy. Add two text lines pulled from
RevenueCat offerings:

- Monthly price string (e.g. "3,99 €/mes")
- Annual price string + savings ("39,99 €/año · ahorra 17%")

Pricing strings come from `offerings.current.monthly.product.priceString` and
`offerings.current.annual.product.priceString`. Savings % computed client-side:
`Math.round((1 - annualPrice / (monthlyPrice * 12)) * 100)`.

Locale-aware (RevenueCat returns localized currency strings per device locale).

### Files touched (Section 4)

- `features/settings/settings.page.html` (PRO hero block)
- `features/settings/settings.page.ts` (compute savings, expose to template)

## Architecture invariants

- All compute lives in pure domain functions (`core/domain/insights/`). Zero Angular deps.
  Easy to unit-test with fixture data.
- State exposed via existing facade pattern (`InsightsStateService`). Dashboard state
  service consumes Insights state read-only.
- No new backend endpoints. No new HTTP calls. All offline-capable.
- Performance: waste + reposition compute lives inside `computed()` signals → recomputes
  only when underlying `events()` or `items()` change. Memoize within compute body if
  events array is very large (> 5k) — measure in pre-flight.

## Risks

| Risk | Mitigation |
|---|---|
| Reposition velocity noisy for users with < 30d history | `confidence: 'low'` items hidden; empty state with "Necesitamos más historial" copy |
| Trial eligibility detection wrong (RevenueCat edge case) | `hasUnusedTrial` returns false → CTA falls back to plain upgrade button |
| Waste tracker shows 0 → looks broken / demotivates | Positive state copy: "¡Sin desperdicios este mes! 🎯" |
| 5 trial CTAs feels spammy | Each surface dismissible per-session via localStorage flag |
| Free-state blurring is unconvincing (user sees value, dismisses CTA) | Show enough to tease, hide exact numbers / item names |
| Currency strings break layout in some locales (de_DE comma, etc.) | RevenueCat returns localized strings; QA in 6 langs before release |

## Success gates (post-ship, +2 weeks)

- ≥ 1 PRO conversion via trial flow (was 0 across v4.6)
- `pro_trial_cta_clicked` fires from ≥ 3 distinct surfaces (proves placement diversity matters)
- `waste_tracker_viewed` and `repo_prediction_viewed` event volumes confirm PRO features are reached
- D7 retention +1–2 pp (umbrella-level, also influenced by Retention sub-spec)
